十年一剑，卷到极致，躺着赚钱的代码随想录

2024-09-29

大家好，我是芋仔。

今天想跟大家聊一聊程序员carl哥，可能名字听起来陌生，但是如果提到代码随想录，我想应该绝大部分的同学都不陌生。

从2020年末开始每天更新一道算法题，到2021星球创立，再到2022星球反哺输出；从算法训练营的稳定运营，到八股训练营，可谓是一步一个脚印，率先实现了知识”正循环”，说是躺着挣钱也不为过。

但是这个钱，分析完carl哥的经历，说实话，活该他挣。

时间线

我们从卡尔哥不多的个人分享中，能够梳理出如下的时间线：

  1. 2010-2014，某工大 c9 硕士，期间一直有准备 acm，但是并没有战绩
  2. 2014-2019, 毕业后先后在BAT工作
  3. 2019年，公众号周更：发布自己的健身经历 (有一说一，carl哥绝对是程序员里最会健身和街舞的哥哥)。
  4. 2020年3月，开始思变，尝试输出面试心得类，个人成长，算法类文章(周更)
  5. 2020年8月，找到了公众号定位，每天输出一道算法题，开启日更，同时宣传自己私人微信号，拉群组队刷题建立私域。
  6. 2021年，算法日更保持，4月份创建知识星球，同时开启春招秋招社招群创建，连接人，建社群。这一阶段，算法，星球，社群互为配合。
  7. 2021年11月，星球反哺，精华帖输出最强八股第一版，2021年12月，代码随想录书籍输出，半价直接卖爆。
  8. 2022年，公众号日更，主要输出为星球产出：面试经验，职场心得，薪资等等，可谓是反哺循环正式启动。
  9. 2022年5月，星球破万，b 站开更视频：算法140题。
  10. 2022年9月，算法训练营启动，主打打卡陪伴答疑，200一位直接行业最低。
  11. 2023年，公众号日更，主要输出为星球产出：面试经验，职场心得，薪资等等，可谓是反哺循环不断上升。
  12. 2023年8月，卡码网上线，主打自己出题，模拟笔试，线上 oj 等等，同步推出 c++，java 基础课，66地板价。
  13. 2024年，公众号日更，4月启动八股训练营，社招训练营

各平台情况

卡尔哥的所有平台，都是一个账号名：代码随想录。

  1. csdn. 最初的平台是 csdn (我知道很多人吐槽，但是其实carl哥现在还会在 csdn更新，流量说实话还可以)。从 2011年开更，而书籍发布是2021，这也是十年一剑的由来。大概450篇文章，6000粉丝。
  2. github. 2020.5 启动，5w start, 7w粉。
  3. b 站. 2020.7 启动，23w粉，27w赞，核心是代码140题详解
  4. 公众号. 核心阵地，2019 开始，2020.8明确定位，一飞冲天。日更算法，到后来的星球精华帖分享，可谓是星球自生长，内循环就足以躺赚。
  5. 掘金. 非核心，100粉，不维护
  6. 抖音. 4000粉，7000赞
  7. 小红书. 4000粉，2022 加入，算法140题

这里面，核心阵地，就是公众号了。

从周更健身素材，到日更算法题，到日更星球精华帖，也是各个训练营，秋招群，星球的宣传主阵地。

作品

那么，从产品角度来说，carl 哥到底有哪些作品呢？我这边整理了一下：

  1. 代码随想录：配套的网站，pdf, 实体书以及b站配套视频课
  2. 最强八股：知识星球中整理的最强八股，需要加入星球获取。但是carl哥也说了，你可以加进去，下载资料，3天内退出就好了， nobody will care。
  3. 卡码网：提供真题模拟，线上oj，设计模式，语言基础课等内容。
  4. 职场问答：星球精华帖整理的内容。

收入分析

carl 哥，由于网站和视频，都是免费的，所以这块并没有直接带来收益，但是间接的促成了后续的成功。

最初的收入，应该是来自于《代码随想录》书籍，当时据说是引爆双十二。不过目前随着网站内容的不断更新，carl 哥也不建议去买书籍了，毕竟无法及时更新。

而之后，就是知识星球了。

这个星球，一方面，加入费用以及续费，是收入来源。目前星球已经有3w多用户，3w乘上任何一个数字，都是一笔非常可观的收入。

而星球，带来的，还有最强八股文，以及随之而来的八股训练营。

同时更重要的，还产出了非常多的职场，薪资，心路，面经等等资料，这部分也成为carl哥公众号的主要输出内容，而这一部分内容，又进一步奠定了星球的新用户及续费意愿，这个正向循环，生生不息。说是躺赚也不为过。

接下来，就是算法训练营了，打卡答疑，对比动辄上千的价格，200的价格可谓是卷到极致了。而每期人数限制，也从150提高到了180，我想实际报名的人数不会少。

除此之外，八股训练营，相对启动时间较晚，但是创新型的录音打卡方式，同时配上极地的价格，极其公开透明的内容，应该会得到不少人的青睐。

而社招训练营，更多的是一种合作模式，价格较高，具体成果不太好衡量。

崛起技巧

但看事实路径总结下来，大概有这么几个技巧

  1. 找准定位
  2. 建立私域
  3. 扩大影响力
  4. 私域运营反哺公域，公域吸引新粉，形成正循环躺赚

初期：找准定位，转换私域，星球直接热启动 (2020.8 - 2021.4)

可以看出来，carl 哥的第一个关键节点，是2020.8.14在公众号的定位：

但这只是开始，找准了定位，宣传公众号，但是这并不能带来收益。

经过一个月的迭代，carl哥的公众号文末的图片改成如下模式：

加入了个人微信，也是所谓私域的开始，通过打卡，聚集自己的用户群体。

这里还有个技巧，你会发现很快，就改了如下模式：

就是增加了自我介绍，如此，方便进一步交流，真正转化为自己”认识”的粉丝。

而这，就组成了他的用户群体，随着每一个同学提问的增多，自然而然，就萌生了星球的想法，而这些用户，就是他星球的种子用户，理所当然的启动了星球，直接带来收益。

就像是生孩子，先自己养，孩子大了，有输出了，再反哺公众号，公众号内容又不断壮大星球，所谓自生长，正循环。

中期：出书/秋招/春招群，免费薅云服务器，进一步扩大影响力 (2021.4-2021.12)

有了正向反馈，接下来就是扩大影响力。

上线代码随想录，出书，建群，在2021.10联动敖丙，免费领云服务器，可以说是极速扩张了。

稳定期：星球反哺，职场/面经/薪资/心得/八股，完成自循环，做更多的事 (2021-2023)

算法写完了，公众号日更从星球的精华帖中产出，也基本是大家关心的问题(毕竟别人付费也想知道的内容，多半是大家都关心的)，从而进一步带动更多的人加入星球，产出更多的内容，又带动新的人加入，这个正循环，简直无敌。

而这时候，carl 哥就相对来说空出来了，可以做更多的事情：算法训练营，卡码网，140题视频更新，扩大影响力的同时，利他又利己，可谓是一举夺得。

成熟期：躺着挣钱(2024)

星球反哺公众号，秋招/春招学生一批一批，公众号又反哺星球，简直躺着赚钱(能够打破这个模式的，我想也只有计算机学生的减少了，或者大家都去考公了)

当然，最核心的，还是要先做利他的事情。

总结

纵观carl哥的一路，从2020.8找到定位后，可谓是一飞冲天，迅速崛起。

这中间，坚持不懈的打卡，持续不断的输出，给应届生，社招带来价值，免费硬核的算法内容，是代码随想录的立身之本。

而可能是无意或者有意通过打卡建立的私域，带来的问答，最终促成知识星球的热启动，是carl哥非常非常关键的一步。大概率也是carl哥不收费持续更新算法的底气(毕竟，没有利益的事情，是真的很难坚持下来)。

而之后秋招，春招，社招群，不断的扩大影响力，最终成就了carl哥的中期成功。

接下来星球的运营，精华帖诞生了最强八股以及各种问答，又反而成为了公众号日更的核心内容，而这些内容，又会反过来带给星球新鲜血脉。至此，正循环形成，生生不息，躺着赚钱。

而之后算法140题的讲解视频，算法训练营的诞生，卡码网的出现，给他人带来价值的同时，又能够得到相应的回报，也是这个模式持续运营的动力来源。

最后，就用carl哥的两句话结尾吧：

前方路艰，与君共勉！

你所期盼，终将到来！

做了半年面试官，我对面试去魅了

2024-02-03

半年前，我第一次成为了二面面试官。这段经历不仅让我，一个曾经的候选人，对许多问题有了更新的理解，也逐渐澄清了我之前在面试过程中的那些困惑。高频次的面试，也使得我对于自己，对于面试这一行为本身，有了深刻的反思和新的认识。我也希望通过分享这些体会，能为那些即将踏上这段旅程的人提供一些启示和帮助。

明明聊的很开心，算法题也做出来了，却没有后续了？

这应该是很多同学在二面的时候遇到的问题。一般来讲，一面完成了基础知识的考察，二面就是你将来的直属上级。

也正是由于二面会是你入职之后的直属上级，这种情况才更容易发生，明明聊的很开心，算法题也做出来了，却没有后续。

从我担任二面面试官的经验来言，因为考虑到候选人入职之后，将是我的直接下属，所以，考察的关键点并不仅仅在于技术层面，性格是否匹配，处事风格如何，过往经历是否匹配，学习能力如何，也是决定是否通过的重点。

除此之外，聊的开心，也并不等于技术过关。

由于工作经验和年限不同，候选人和面试官，往往在技术视野上存在一定的 gap。而项目经验的不同，也会导致候选人和面试官在技术的了解上存在偏差，双方熟悉的领域并不相同。

碰到有类似的项目经验的面试官，那么在深度上，往往会深挖，这一过程中，候选人可能会感觉良好，觉得问对了，觉得聊的比较开心。

而在面试官的角度，可能并不会认同，由于技术视野和技术深度的偏差，一旦候选人达不到预期的深度或方案上没有自己的思考，可能会认为候选人在技术方案和深度上，都有待加强。

当然，除此之外，还有很多原因会导致没有后续(不过 hc 不足，有更好的候选人等，这些其实一般不会发生在二面，往往会发生在 hr 面之后)，上述也仅仅只是我个人作为面试官的一些想法。

这面试官问的问题怎么这么奇怪？这问题有答案吗？

不知道有没有同学遇到过，面试官在聊了项目之后，会抛出一个非常开放的问题，有可能是项目的延伸，有可能是项目的未来规划，也有可能是一个不好实现的问题。

比如：如何在 vue 里面使用 react 组件，react 能够使用 vuex 吗这种天马行空的问题 (当然，这个问题纯属示例，一般都会和项目相关，比如如果重构，你会多出哪些考虑，能不能用另一种方案实现它)。

这里的背后，作为面试官一般是这样的心理活动，和候选人这个项目聊的还不错，深度也还 ok，姑且问个开放性的问题，看看随机应变的能力和创新性吧。

遇到这样的情况，其实也很简单了，面试官要的并不是一个准确的答案，甚至有可能问题本身就有问题。

此时，就大开脑洞，也别管能不能实现，正确与否，按照你的想法，天马行空也好，讨论问题也罢，聊就对了。

你有遇到过什么困难或者有技术难度的问题吗？

我自己作为候选人被面试的时候，有时候会被问到这个问题。

当时的内心 os 真的是，你问这么虚的问题，真不靠谱，敢不敢问点实在的。

当我成为面试官之后，我才明白，这个问题背后的意思（当然，这仅仅代表部分情况并非全部）。

往往会在聊了一会项目之后，问出这个问题。

这里其实要注意了，问出这个问题，大概率意味着，前面的项目咱们都聊过了，在我的视角下，可能不是太有挑战性，技术上复杂度一般，当然，也可能是因为咱们项目经验的差别，导致我看不出你做的项目的复杂度。

此时，问出这个问题，让候选人自己把困难或者项目的复杂度抛出来，来避免由于我自己的判断导致错误估计了候选人的技术实力。

一般一到两年经验的同学，被问到这个问题还算正常，因为一般来讲，这个年限的同学做的事情都比较基础。而如果是三到五年的同学，那么就需要看看是否自己简历上的项目描述是否有突出复杂性和难度。

要应对这个问题，就必须上点技术复杂度了。如果业务复杂，可以讲述业务的情况和场景，如果技术复杂，那么可以看看有哪些你认为复杂的点没有聊到，可以再聊聊。

如果业务确实不复杂，那么可以聊聊看过的源码，或者自己平时钻研的一些问题或者当下热门的话题，比如之前 react18 的 renderToPipeableStream 和传统 ssr 的区别，或者前一段时间 vue 作者认为 react 文档存在让开发者自行 pua 的情况的讨论。

这里的核心，就在于突出自己的技术深度以及对各种方案背后的思考。

那到底什么算深度呢？

提到深度，如果是以前的我，必然会反驳，前端，就是画画页面，有什么深度可以说呢？

但是实际上，大家还是低估了自己工作的复杂度，画页面的简单，仅仅只是因为那些复杂的东西，其他同学在底层都处理掉了。

在一个成熟的前端团队中，工具层面的复杂度基本都在底层被一些同学或者一些组进行了解决。

比如：资源文件的缓存处理，html 文件一般进行协商缓存或者不缓存，而 js、css 文件一般使用强缓存。这些在部署平台就进行了处理，从我面试的候选人来讲，知道协商缓存强缓存的很多，涉及的几个头背的滚瓜烂熟，但是他们的具体用途，往往会含糊不清有所犹豫。

再比如组件库，基本上大家都在用，那么组件库是如何实现按需加载的？或者你的项目中，组件库是否是正常按需加载的吗？组件库的组件，应该提供哪些打包产物呢？你的项目中又是什么样的产物呢？对于样式文件，有哪些处理方案呢？各自的优劣如何？

再比如，项目的打包产物，是否还需要对 promise, async/await 这些语法进行 polyfill，2024 年了，我们的 babel 还有必要把 es6 语法转为 es5 吗？

而更底层的复杂度，从 Jquery, 到 react, vue，再到 svelte, solid 等声明式框架，被这些更底层的框架所掩盖。

那么回到深度，对于做业务的同学，确实不太会对构建打包，组件库、脚手架建设有那么多的了解。但是业务同学，是需要对整个项目的技术选型，性能优化，代码可维护性负一定的责任的。

这里我也只能结合我自己的经验聊一聊深度，并不具有代表性。

比如常见的 react 中的多次渲染如何优化，为什么需要全局状态管理，context 如何避免不必要的渲染，对于 redux 有多少了解，它的中间件思想可以拿出来用到项目中吗？这是我在给组内设计一款合适的状态管理时做的思考，在后来准备面试的时候就做了更多相关的了解。

再比如一些项目本身，就有一定的复杂度，比如我之前做的可视化埋点平台，涉及到一些生僻 api 使用，iframe 通信的设计，以及 babel 插件、webpack 插件等来稳定 xpath 的开发。而这个技术项目，本身也是在我们运营活动的场景下所演化出来的一个平台。

这里建议候选者，对自己的项目一定要进行深挖，涉及到的相关知识点，务必要吃透。比如写了在用 redux，那么势必就需要对其原理有比较多的了解。

本身面试双方信息就是不对等的，面试官一旦在你的优势项目中，发现深入度和思考还不如自己初看这个项目时的思考，将会是非常大的一个减分项。

结语

以上，便是我的一些感悟和思考。

其实这半年的时间，于我而言，也可以说是一场对面试去魅的过程。

站在面试官的角度去实际同候选人交流，有时候就像遇到过去的自己。也很快就理解了以前的我，单纯作为候选人的时候的诸多困惑。

希望本文的一些解答，能够帮助大家对与面试有更好的理解。也祝大家在新的一年里，都有更好的 offer！

解决方案

经过对vscode源码debug，基本了解了大致流程。得到的正确解法如下：

解法1:

删除项目目录下的jsconfig.js，可以实现jsx，js文件的自动跳转

解法2:

jsconfig.json中配置如下：

{
    "compilerOptions": {
      "jsx": "preserve", // 共有三种枚举： react, react-native, preserve，实测均可
   },
    "exclude": ["node_modules"]
}

改进解法2:

一般来讲，jsconfig.json的存在都是配合webpack alias进行快速跳转，故可以参考如下配置：

{
    "compilerOptions": {
      "jsx": "react-native",
      "baseUrl": "./",
      "paths": {
        "@/*": ["src/*"]
      }
    },
    "exclude": ["node_modules"]
}
